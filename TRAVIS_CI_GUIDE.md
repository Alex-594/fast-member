# Инструкция по сборке APK через Travis CI

## Что такое Travis CI?

Travis CI - это облачный сервис для автоматической сборки и тестирования проектов. Он работает похоже на GitHub Actions, но может быть проще для некоторых задач.

## Преимущества Travis CI:

- ✅ Работает на Linux из коробки
- ✅ Простая конфигурация
- ✅ Бесплатно для публичных репозиториев
- ✅ Хорошо работает с Buildozer

## Как использовать:

### Шаг 1: Регистрация на Travis CI

1. Перейдите на https://travis-ci.com (или https://travis-ci.org для публичных репозиториев)
2. Войдите через GitHub
3. Разрешите доступ к вашим репозиториям

### Шаг 2: Создание репозитория на GitHub

1. Создайте новый репозиторий на GitHub
2. Загрузите ваш проект (включая файл `.travis.yml`)

### Шаг 3: Активация Travis CI

1. Перейдите на https://travis-ci.com
2. Найдите ваш репозиторий в списке
3. Включите его (переключите тумблер)

### Шаг 4: Запуск сборки

1. Сделайте коммит и push в репозиторий
2. Travis CI автоматически запустит сборку
3. Дождитесь завершения (30-60 минут)

### Шаг 5: Скачивание APK

1. После завершения сборки перейдите на страницу сборки
2. В разделе "Artifacts" или "Download" будет APK файл

## Важные замечания:

- Файл `.travis.yml` уже создан в вашем проекте
- Travis CI автоматически запустится при каждом push
- Для публичных репозиториев - бесплатно
- Для приватных репозиториев может потребоваться платная подписка

## Альтернатива: GitHub Actions

Если Travis CI не подойдет, можно вернуться к GitHub Actions - там тоже можно настроить сборку, но конфигурация может быть сложнее.
